# ✅ Criterios de Aceptación (Given / When / Then)

## 2 – Inicio de sesión

**Given** que el usuario está registrado en el sistema  

**When** el usuario ingresa su correo y contraseña correctamente  

**Then** el sistema le permite acceder a su cuenta

---

## 4 – Agendar cita

**Given** que existen horarios disponibles  

**When** el cliente selecciona una fecha y hora  

**Then** el sistema registra la cita en la agenda

# 🚀 MVP Rationale

El **Minimum Viable Product (MVP)** del sistema incluye las funcionalidades esenciales que permiten que el sistema funcione desde su primera versión.

Estas funcionalidades incluyen:

- Registro de usuario
- Inicio de sesión
- Visualización de horarios disponibles
- Agendamiento de citas

Estas características permiten que los clientes puedan reservar citas y que el negocio pueda gestionar su agenda.

Las demás funcionalidades, como notificaciones o gestión avanzada de servicios, se consideran mejoras futuras que pueden implementarse en versiones posteriores del sistema.
